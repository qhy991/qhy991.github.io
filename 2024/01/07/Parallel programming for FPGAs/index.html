Array Partition 和 Array Reshape的区别

在FPGA设计中，数组重塑（array reshape）和数组分区（array partition）都是用来增加每个时钟周期可以读取的数组元素数量的技术。这两个指令都支持相同的分区选项，允许循环和块分区，或者沿着多维数组的不同维度进行分区。它们的主要区别在于对地址空间的处理方式及其对硬件资源的映射效率。

数组分区 (Array Partition)

  • 灵活性：数组分区更加灵活，因为它可以将数组的不同部分映射到FPGA上的不同物理内存块中。

  • 地址空间：分区后的数组，每个分区可以有不相关的地址空间。

  • 内存大小：分区导致每个独立的内存变小，可能导致内存使用效率降低，尤其是当分区小于单个BRAM块的大小时。

数组重塑 (Array Reshape)

  • 内存块大小：数组重塑通常会导致更大的内存块，有时可以更有效地映射到FPGA的基本资源中。

  • 地址空间保持：重塑后的数组元素必须保持相同的地址，即在转换后的数组中，元素的地址空间是连续的。

  • BRAM使用：在Xilinx Virtex Ultrascale+设备中，BRAM的最小粒度是18 Kbits，可以支持多种深度和宽度的组合。如果分区的数组小于大约18Kbits，那么BRAM就不能高效使用。

以一个4位宽、维度为[1024][4]的原始数组为例，这个数组可以适配到一个配置为4Kbit x 4的单个BRAM资源中。如果在第二维完全分区这个数组，将得到4个1Kbit x 4的内存，每个都比一个BRAM资源小很多，这样的利用率是不高效的。相反，如果使用数组重塑指令，将得到一个1Kbit x 16的内存，这是一个支持的BRAM配置。

选择哪种技术

选择使用数组分区还是数组重塑，需要根据具体的设计要求和资源限制来决定。如果需要更大的并行性和灵活性，且对于单个内存的大小没有严格要求，数组分区可能是更好的选择。但如果要保证内存使用的效率和映射到FPGA资源的效率，特别是在资源受限的情况下，数组重塑可能更加适合。

在实际应用中，设计者需要评估数组的大小、访问模式和FPGA的资源情况，然后选择最适合的方案。例如，对于较大的数组，可能需要使用数组分区来提高并行性；而对于较小的数组，使用数组重塑可能更能有效利用BRAM资源。此外，还要考虑实现复杂度和可能的性能权衡。在Vivado HLS等工具中，可以通过相应的指令（例如#pragma HLS ARRAY_PARTITION 和 #pragma HLS ARRAY_RESHAPE）来实现这些技术。

Chapter 6 Sparse Matrix Vector Multiplication

引入了一个更加复杂的数据结构， compressed row storage（CRS），来存储sparse matrix

columnIndex 存储的是每一个原属的column

rowPtr 存储的是每一行第一个元素的index

从一个CSR的数据结构中，还原回二维数组，首先根据rowPtr来决定第i行都有几个NZ。 rowPtr[i] - rowPtr[i+1].然后就可以索引value中的元素，values[rowPtr[i]],这个是第i行第一个NZ

如果是矩阵和向量进行相乘的话，就不需要将CSR的矩阵还原回2维矩阵形式。

对于那些执行时间是data depedent的情况，HLS 不能给出性能估计，这个时候我们可以给HLS提供一些信息，使用 loop_tripcount ，指明循环的最大，最小和平均iteration

当循环边界是不确定的时候，也不能使用unroll来优化

Chapter 7 Matrix Multiiplication

array_partition 和 array_reshape的区别

array_partition 和 array_reshape都能增加一个时钟周期读到的数据量。

array_reshape，元素在转换后的array中必须用相同的地址。

arrayy_partition，转化后的array的地址可以是不相关的。

虽然partition更加灵活，它使得每一个单独的存储变得更小，但这会导致存储利用的低效。

Ultrascale+的FPGA中能够控制BRAM的最小粒度是18 Kbits，能够支持深度和宽度的不同组合。

当一个array的partition的尺寸比18Kbit还要更小的话，那么就会造成利用的低效。

举个例子，一个4bit的数组，[1024][4]，如果使用partition，那么要使用4个 1Kbit x4 的memory，reshape的话，会按照1kbit x 16的方式存储。

Streaming architecture

一次传输一块数据而不是所有数据。

还有一个优点就是，数据即来即用不需要保存，能够节省memory
